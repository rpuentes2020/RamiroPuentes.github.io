Project Overview

The scope for this project consisted of upgrading an existing access database application which relied on access forms.

The analyst for this department scoped a request to upgrade the existing application which was designed by staff on premise using Microsoft Access forms.

Project duration August 2016 - October 2016

My role in the project was that of a UX UI Designer, Database Designer and Developer.

User Research

The existing application did not provide any type of security. It was running on MS Access forms connected to a Sql backend. The existing database design did not incorporate any Keys or Id's which resulted in years worth of duplicate records for users, vehicles, permits, and lots.

Pain Points

Data

The existing database consisted of flat tables with no keys or Id's. Many duplicate entries, records, permits and lots.

Security

The application didn't provide any type of security.

Frustration

Staff were frustrated using the current application, because of inconsistencies and duplicate records.

Address Issues

Data Clean Up

It took several weeks to sanitize the data, but I was able to get a workable dataset prepared for migration.

Reports

The department was provided with reports to review and decide how edge cases were to be handled.

Fresh Start

A new database design was created to follow a friendly user flow for both users and administrators.

Launch

The updated application allowed staff to access the application from any Fresno County Access point, allowed several new features; accessible forms, reports, easy user lookup, and the ability to create new permit or parking types.

Take Aways

Impact

Updating this application resulted in better UX UI for administrators and users. This also allowed better management of permits, lots and assignable stalls.

What I Learned

I learned several ways to sanitize data from an almost hopeless record set. It took several attempts but I managed to recover enough core data to prevent manual entry by administrators.

Project Overview

I was asked to make a query update to the Public Health Inspections website. What I saw shocked me. I couldn't make a query change an ignore the design of the website.

The department submitted a request to update the search query for their site and didn't intend on paying for any other changes. Knowing my time was limited I quickly put together a proposal which included; some designs, a cost and time estimate, and shared it with the department. They loved the idea and jumped at the opportunity.

Project duration October 2017 - October 2017

My role in the project was that of a UX UI Designer and Developer.

User Research

The existing website was not accessible and suffered from long page loads caused by inefficient data requests. The department identified the slowness and requested a specific change to the search query in hopes that would speed up the page. After reviewing the query, I realized the workflow could be expanded which would offer a better user experience and parse the data request into smaller requests.

Pain Points

Slow

The original query was pulling unnecessary data that wasn't displayed within the requesting view

Mobile

The current site didn't offer any accommodations for a mobile experience.

Contact Info

The current site did not provide any contact information for the searched entity.

Address Issues

Design

The department was ecstatic with the opportunity for a redesign of the site

Accessibility

Update the site to be responsive, and accessible.

New Features

Add the ability to use google maps to get directions or call the entity from your mobile device.

Launch

Updates to the workflow resulted in smaller more efficient queries. The new workflow allowed me to split the large query into multiple smaller requests. The new workflow also allowed users to browse locations before selecting additional information pertaining to a specific location

Public Health Inspections (Additional changes have been made by customer since launch)

Take Aways

Impact

The updates I made to the site made it responsive, mobile-friendly and accessible. Adding a workflow that users understood and features that incorporated directions using Google Maps

What I Learned

This helped me explore the available features a website has when visited on a mobile device. I enjoyed providing users with additional features, and the department was excited about the new design as well.
